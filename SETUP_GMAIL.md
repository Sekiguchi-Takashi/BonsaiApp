# Gmail 連携のセットアップ（無料・Google Cloud プロジェクト不要）

OAuth は使わない。**アプリパスワード + IMAP** なので、
Google Cloud のプロジェクト作成もアプリ審査もトークン期限切れも無い。

## 前提

- **個人の gmail.com アカウント**であること
  （Google Workspace はアプリパスワードによる IMAP が塞がれている場合が多い）
- **2段階認証が有効**であること（アプリパスワードの発行条件）
- **Advanced Protection に登録していない**こと（登録するとアプリパスワードが使えない）

IMAP の有効化は不要。2025年1月以降は既定でONになっている。

## 手順

1. スマホのブラウザで https://myaccount.google.com/apppasswords を開く
   - アプリ内の「アプリパスワードを発行する」ボタンからも開ける
   - このURLからしか入れない（設定画面上の導線は撤去されている）
2. 2段階認証が未設定なら、先に有効化を求められる
3. 名前を入力（`Bonsai` など）して作成
4. **16桁のパスワードが表示される。この画面を閉じると二度と見られない**
5. アプリの「メール連携」画面に、Gmailアドレスと一緒に貼る
   - 空白は入れても入れなくてもよい（内部で除去する）
6. 「メール監視を開始」

## 差出人フィルタ

空欄なら受信した未読メール全件が対象。
特定の相手だけ拾いたい場合はカンマ区切りで指定する（部分一致）。

```
boss@example.com, @mycompany.co.jp
```

## 動作の仕組み

- IMAP **IDLE** で待ち受けるので、ポーリングしない。
  新着があった瞬間にサーバー側から通知が来る
- Gmail は29分で IDLE を切るため、24分ごとに張り直す
- 切断されたら5秒→10秒→…→最大5分の指数バックオフで再接続
- **`BODY.PEEK[]` で取得するので既読フラグは立たない。**
  Gmail 側の未読状態を汚さない
- 取得済みの UID を記録しているので、同じメールを二重に積まない

## 注意点

- **アプリパスワードは SharedPreferences に平文で保存される。**
  端末が root 化されていなければ他アプリからは読めないが、暗号化はしていない
- Googleアカウントのパスワードを変更すると、アプリパスワードは失効する
- 添付ファイルは取得しない。本文（text/plain、無ければ text/html をタグ除去）のみ
- 本文は 20,000 文字で切り捨てる

## 動作確認

1. 「メール監視を開始」→ ステータスが「待機中 (IDLE)」になる
2. 自分宛にテストメールを送る
3. 数秒でキューに現れる

現れない場合、まず「受信位置をリセット」を押してから再確認する。
それでもダメならログを見る:

```
logcat -d | grep -i -E "MailWatcher|ImapClient" | tail -30
```

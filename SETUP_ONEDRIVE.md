# OneDrive 連携のセットアップ（1回だけ・無料）

Microsoft Graph を使うには **クライアントID** が要る。
Azure のアプリ登録は個人アカウントで無料、クレカ登録も不要。
スマホのブラウザだけで完結する。

## 手順

1. https://portal.azure.com を開き、Microsoftアカウントでサインイン
2. 検索窓に「アプリの登録」→ **新規登録**
3. 入力内容:
   - **名前**: `BonsaiRAG`（何でもよい）
   - **サポートされているアカウントの種類**:
     「任意の組織ディレクトリ内のアカウントと個人の Microsoft アカウント」
   - **リダイレクト URI**: **空のまま**（device code flow なので不要）
4. 登録後の「概要」に出る **アプリケーション (クライアント) ID** をコピー
5. 左メニュー **認証** → 下の方の
   「パブリック クライアント フローを許可する」を **はい** にして保存
   - ここを忘れると device code flow が使えない。**一番よくある詰まりどころ**
6. アプリの「RAG設定」画面にクライアントIDを貼り、フォルダ（既定 `/RAG`）を確認
7. **サインイン** を押す → ブラウザが開くのでコードを貼って承認

API 権限の追加設定は不要。`Files.Read` は device code flow の
同意画面でその場で承認される。

## フォルダ指定

OneDrive のルートからの相対パスで書く。

- `/RAG` → マイファイル直下の `RAG` フォルダ
- `/Documents/RAG` → `Documents` の下の `RAG`
- サブフォルダも再帰的に同期される

## 対象ファイル

`.txt` / `.md` / `.markdown` / `.text` のみ。
2MB を超えるファイルはスキップする。
PDF や Office ファイルは対象外（依存ゼロを維持するため）。

## 差分同期の仕組み

初回はフォルダ全体を取得し、Graph が返す `deltaLink` を保存する。
2回目以降はその `deltaLink` を叩くので、**変更があったファイルだけ**が返る。
ファイルを1つ直しただけなら通信は数KBで済む。

削除された分は `deleted` フラグ付きで返るので、インデックスからも消える。

## トークンの保存場所

`SharedPreferences`（アプリ専用領域）に平文で保存している。
端末が root 化されていなければ他アプリからは読めないが、
暗号化はしていない。`EncryptedSharedPreferences` は androidx 依存になるため
今回は見送った。気になる場合は「サインアウト」でトークンを消せる。

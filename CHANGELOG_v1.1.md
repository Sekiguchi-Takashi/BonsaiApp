# v1.1 — トップ画面から RAG を使えるようにした

## 背景
トップ画面の入力欄はモデル単独で答える実装だったため、
「結婚祝金はいくらですか」と聞くと資料を見ずに作話していた。
仕様どおりの動作ではあるが、**RAGの検証ができない**という問題があった。

## 変更

### トップ画面に RAG トグルを追加
- **RAG参照: ON** — 質問で資料を検索し、上位チャンクを文脈に積んで回答する。
  回答末尾に参照ファイル名を表示（資料が無ければ「参照した資料なし」）
- **RAG参照: OFF** — 従来どおりモデル単独

既定は ON。

### Pipeline を汎用化
メール処理専用だった `Pipeline` から `retrieve()` と `answer()` を切り出し、
トップ画面・メール処理・APIの3経路で共用するようにした。
文脈の予算管理（資料1100文字 / 質問600文字 / 回答400トークン）は共通。

### API でも RAG が使える
リクエストに `"rag": true` を足すと資料検索を挟む（OpenAI仕様の独自拡張）。

```
curl http://127.0.0.1:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{"model":"bonsai","rag":true,
       "messages":[{"role":"user","content":"経費精算の期限は"}]}'
```

### 表記の修正
ボタン名を「RAG設定 (OneDrive)」→「RAG設定 (資料フォルダ)」に変更。
v0.8 でクラウド直結をやめているのに古い名前が残っていた。

## テストへの影響
`TEST_PLAN.md` のレベルA〜E を、**メールを送らずにトップ画面から試せる**ようになった。
検証がだいぶ速く回せるはず。

## 注意
RAG参照 ON にすると、資料1100文字ぶんのプリフィルが毎回入る。
4B だと最初のトークンが出るまで10〜20秒かかることがある。
tok/s の数字自体は生成部分のみを見ているので、体感より良い値が出る。
